## 功能概览

| 功能 | 简称 | 一句话描述 |
|------|------|-----------|
| 稿件润色模式 | Polish | 用户写一份粗稿（大纲/要点/草稿），AI 帮润色排版为完整公众号文章 |

## 一、稿件润色模式（Polish）

### 1.1 用户场景

用户已经有了自己的想法甚至粗稿，但不想从零让 AI 写——而是希望 AI 基于自己的内容进行润色、扩写、排版。

**典型使用方式：**

- 「帮我把这几个要点扩写成一篇完整文章」
- 「我写了个大纲，帮我润色排版，加上过渡段和 CTA」
- 「这是我写的初稿，帮我改得更有公众号的感觉」

### 1.2 与现有「创作模式」的区别

| | 创作模式（现有） | 润色模式（新增） |
|--|-----------------|-----------------|
| 输入 | 主题 + 补充说明（简短） | 用户粗稿（大段文本） |
| AI 角色 | 从零创作者 | 编辑 / 润色者 |
| 用户控制度 | 低（AI 自由发挥） | 高（基于用户原始内容） |
| 输出 | 全新文章 | 保留用户核心观点，优化表达和排版 |

### 1.3 交互设计

#### 模式切换

在创作页顶部、输入卡片内、标题行下方新增模式切换 Pill：

```
[✨ 创作] [✏️ 润色]
```

- 默认选中「创作」，即当前的完整生成流程
- 点击「润色」后输入区变化（见下方）

#### 润色力度说明

| 力度 | AI 行为 | 适合场景 |
|------|--------|---------|
| 轻度润色 | 只改错别字、调语气、加标点分段、套公众号排版格式。不改动内容结构 | 初稿已经比较完整 |
| 中度改写 | 优化表达、补充过渡句、加 emoji 增强可读性、增加小标题。可能合并或拆分段落 | 有大纲和要点，需要充实 |
| 深度扩写 | 大幅扩写要点为完整段落、补充案例/数据/金句、加 CTA。输出接近全新文章，但保留用户核心观点 | 只有几个关键词或大纲 |

```javascript
const POLISH_LEVEL_MAP = {

    - 修正错别字和语法问题
    - 调整语气，使其更适合公众号阅读
    - 添加合适的标点和分段

    - 不要自行添加原文中没有的信息`,

  '中度改写': `你是一位公众号内容编辑。用户提供了大纲或初稿，请你：
    - 优化每个段落的表达，使其更生动流畅
    - 补充适当的过渡句，让文章连贯
    - 在合适的位置添加 emoji 增强可读性
    - 提炼 2~4 个小标题（<h2>）组织内容
    - 可以合并或拆分段落以改善阅读节奏

  '深度扩写': `你是一位公众号内容创作者。用户提供了要点或大纲，请你：
    - 将每个要点扩写为完整的段落（含案例、数据或场景描写）

    - 设计 2~4 个小标题，构建清晰的文章结构
    - 添加金句（<div class="hl">）和行动号召（<div class="cta">）
    - 补充 AI 认为有价值的相关信息
    - 用户原始要点必须全部覆盖，不可遗漏`,
};

const PRESERVE_MAP = {
  '保留原文核心观点': '必须保留用户原文中的所有核心观点，不可替换或删除',
  '保留原文关键用词': '用户原文中的品牌名、产品名、专有名词必须原样保留',

  '允许AI调整段落顺序': '如果调整段落顺序能让文章更流畅，可以重新排列',

```

## 二、风格学习系统（StyleBank）

### 2.1 用户场景

用户看到某些优秀的公众号文章，希望 AI 能学习这些文章的写作风格，后续生成时带上相同的风格调性。

**典型使用方式：**

- 「我喜欢这个公众号的写法，帮我学一下它的风格」
- 「这三篇文章都是我想要的感觉，记住这种风格」
- 「用我之前保存的『文艺清新风』生成这篇文章」

### 2.2 完整使用流程

```
第一步：喂文章
用户粘贴或上传 1~3 篇参考文章 → AI 分析提取风格特征

第二步：确认风格卡片
AI 输出风格摘要（语气、句式、用词偏好、结构特点等）
→ 用户确认或微调 → 保存为一张「风格卡片」

第三步：应用风格
创作或润色时，用户从已保存的风格卡片中选择一张
→ AI 生成时自动带入该风格的写作指令
```

### 2.3 风格卡片（StyleCard）数据结构

```javascript
{
  id: 'style_1714444800_abc',
  name: '文艺清新风',          // 用户可自定义命名
  description: '',             // AI 生成的风格摘要（展示用）
  createdAt: '2026-05-01T10:00:00Z',
  sourceCount: 2,              // 学习了几篇文章
  
  // AI 提取的风格维度
  features: {
    tone: '温暖亲切，像朋友聊天',
    sentenceStyle: '短句为主，偶尔用长句制造节奏感',
    vocabulary: '偏口语化，常用「超级」「绝了」「谁懂」等网络用语',
    structure: '开头用反问或场景切入，中间分 3~4 个小节，结尾金句收束',
    emoji: '频繁使用，每段 1~2 个，偏好食物和表情类 emoji',
    rhetoric: '善用比喻和排比，喜欢用数字制造冲击感',
    paragraph: '段落短小，每段 2~3 句，留白多',
    cta: 'CTA 语气轻松，不用「立即购买」，而是「去试试吧」',
  },
  
  // AI 生成的可直接注入 Prompt 的风格指令
  promptInjection: '请使用以下写作风格：语气温暖亲切如朋友聊天...（完整指令文本）',
}
```

#### 创作/润色时选用风格

在创作页的输入卡片中，模板选择下方新增一行：

```
── 写作风格（可选） ────────────────────
[默认]  [文艺清新风]  [专业权威]  [活泼种草]
```

- 「默认」= 不注入任何风格指令，使用品牌设置中的「写作语气」
- 选中某风格卡片后，生成时该风格的 `promptInjection` 会被注入到 Prompt 中

### 2.5 风格分析 Prompt

用于让 AI 分析参考文章、提取风格特征：

```javascript

【参考文章】
${articles.map((text, i) => `--- 第 ${i + 1} 篇 ---\n${text}\n`).join('\n')}

{
  "name": "用 4~6 个字概括这种风格（如：文艺清新风、硬核测评风、温暖治愈风）",
  "description": "用 1~2 句话描述这种风格的整体感觉",
  "features": {
    "tone": "语气和态度（如：亲切、权威、幽默、冷静）",
    "sentenceStyle": "句式特点（如：短句为主、长短交替、排比多）",
    "vocabulary": "用词偏好（如：口语化、书面正式、网络热词多）",
```

### 2.7 存储设计

风格卡片存储在 localStorage 中：

```javascript
const STYLES_KEY = 'wechat_tool_styles';

function loadStyles() {
  try { return JSON.parse(localStorage.getItem(STYLES_KEY)) || []; }
  catch { return []; }
}

function saveStyle(styleCard) {
  const styles = loadStyles();
  styles.unshift(styleCard);
  if (styles.length > 10) styles.splice(10); // 最多 10 种
  localStorage.setItem(STYLES_KEY, JSON.stringify(styles));
}

function deleteStyle(id) {
  const styles = loadStyles().filter(s => s.id !== id);
  localStorage.setItem(STYLES_KEY, JSON.stringify(styles));
}
```

## 四、开发优先级建议

| 顺序 | 功能 | 理由 |
|------|------|------|
